## Architecture reminder

The app is a single Python process: FastAPI + APScheduler (pollers + workers) all in-process.
Entry point: `python -m trumptrade` (defined in pyproject.toml `[project.scripts]`).
Uvicorn is imported internally via `uvicorn.run()` in `trumptrade/__main__.py`.
SQLite lives at `./trumptrade.db` relative to the working directory.
Frontend is a Vite/React SPA — built to `frontend/dist/`, served as static files.

## Oracle Cloud Free Tier Instance — Step by Step

**Cost:** Free forever (Always Free tier). No credit card charge as long as shape is VM.Standard.A1.Flex.

### 1. Create an Oracle Cloud account
- Go to https://cloud.oracle.com and sign up.
- Choose your home region carefully — it cannot be changed. Closest region to you is fine.
- You must verify identity with a credit card (charged $0 if you stay in Always Free limits).

### 2. Create a Compute Instance
1. In the OCI Console: **Compute → Instances → Create Instance**
2. Name: `trumptrade`
3. **Image:** Click "Change image" → select **Canonical Ubuntu** → version **22.04 (aarch64)**
4. **Shape:** Click "Change shape" → Ampere → **VM.Standard.A1.Flex**
   - OCPU: **1**
   - Memory: **6 GB** (you get 24 GB total free; use 6 here, save 18 for future)
5. **Networking:** Accept default VCN or create one. Ensure "Assign a public IPv4 address" is checked.
6. **SSH keys:** Upload your existing public key (`~/.ssh/id_rsa.pub`) OR let Oracle generate one (download it immediately).
7. **Boot volume:** Default 50 GB is fine (Always Free includes 200 GB total).
8. Click **Create**.

Wait ~2 minutes for status to show **Running**.

### 3. Note your public IP
- On the instance detail page, copy the **Public IP address**. You'll use it in all subsequent tasks.

### 4. Open ports 80 and 443 in the OCI Security List
**This step is required — OCI has a separate network firewall layer that UFW alone does not control.**

1. In the instance detail page, under "Primary VNIC", click the **Subnet** link.
2. Click the **Security List** (usually named "Default Security List for...").
3. Click **Add Ingress Rules** and add these two rules one at a time:

   **Rule 1 — HTTP:**
   - Source Type: CIDR
   - Source CIDR: `0.0.0.0/0`
   - IP Protocol: TCP
   - Destination Port Range: `80`
   - Description: `HTTP for Nginx`

   **Rule 2 — HTTPS (future use):**
   - Source Type: CIDR
   - Source CIDR: `0.0.0.0/0`
   - IP Protocol: TCP
   - Destination Port Range: `443`
   - Description: `HTTPS`

4. Click **Add Ingress Rules** to save.

**Why this matters:** Oracle Cloud puts a hardware-level Security List in front of the VM's own firewall (UFW). If this step is skipped, port 80 traffic never reaches the VM even if UFW is open.

### Alternative: DigitalOcean or Hetzner ($4-6/month)
If Oracle Free Tier signup fails or quota is unavailable in your region:
- **DigitalOcean:** Create Droplet → Ubuntu 22.04 → Basic → $4/mo (512 MB RAM) or $6/mo (1 GB RAM, recommended). x86_64.
- **Hetzner Cloud:** Create server → Ubuntu 22.04 → CX11 (€3.79/mo, 1 vCPU, 2 GB RAM). x86_64. Best value if available in your region.
- For DigitalOcean/Hetzner: No separate Security List step — UFW alone is sufficient.

### 6a. Create deploy/trumptrade.service

Create directory `deploy/` in the repo root. Write the file:

```ini
[Unit]
Description=TrumpTrade Bot
Documentation=https://github.com/YOUR_USERNAME/trump_trade
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=trumptrade
Group=trumptrade
WorkingDirectory=/home/trumptrade/trump_trade
EnvironmentFile=/home/trumptrade/trump_trade/.env
ExecStart=/home/trumptrade/trump_trade/.venv/bin/trumptrade
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal
SyslogIdentifier=trumptrade
# Hardening
NoNewPrivileges=yes
PrivateTmp=yes

[Install]
WantedBy=multi-user.target
```

**Key fields explained:**
- `EnvironmentFile=` — loads .env secrets into the process environment
- `WorkingDirectory=` — SQLite `./trumptrade.db` resolves here (but we set DB_URL to absolute path in .env anyway)
- `Restart=always` — restarts on any exit including crashes
- `RestartSec=10` — waits 10s before restart (prevents tight crash loops)
- `ExecStart=` — uses the venv's installed entry point binary, not `python -m trumptrade`

### 6b. Create deploy/nginx.conf

```nginx
server {
    listen 80;
    server_name _;  # Matches any hostname — replace with domain if you have one

    # API and WebSocket proxy — FastAPI on :8000
    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 86400s;  # Long timeout for WebSocket connections
    }

    location /ws/ {
        proxy_pass http://127.0.0.1:8000/ws/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 86400s;
    }

    # React SPA static files
    location / {
        root /home/trumptrade/trump_trade/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;  # Client-side routing fallback
        expires 1h;
        add_header Cache-Control "public, max-age=3600";
    }
}
```

**Key design decisions:**
- `/api/` and `/ws/` routes proxy to FastAPI — all other routes serve the React SPA
- `try_files $uri $uri/ /index.html` — React Router client-side navigation works correctly (no 404 on refresh)
- `proxy_read_timeout 86400s` — WebSocket connections stay alive for 24 hours
- `Upgrade` and `Connection` headers — required for WebSocket protocol upgrade
