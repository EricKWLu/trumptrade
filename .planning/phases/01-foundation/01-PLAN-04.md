Run from the project root (`c:/Users/ericl/Projects/trump_trade/`):

```bash
alembic init --template async alembic
```

CRITICAL: Use `--template async`. Plain `alembic init alembic` generates a sync env.py that fails with `sqlite+aiosqlite://` — it cannot use an async driver.

After the command completes, you will have:
- `alembic.ini`
- `alembic/env.py`
- `alembic/script.py.mako`
- `alembic/versions/` (empty directory)

**Step 2 — Edit `alembic.ini`**: Find the line `sqlalchemy.url = driver://user:pass@localhost/dbname` and replace it with:
```
sqlalchemy.url = sqlite+aiosqlite:///./trumptrade.db
```

Also ensure `prepend_sys_path = .` is set (the async template includes this; verify it is present).

**Step 3 — Edit `alembic/env.py`**: The async template generates a file with placeholder `target_metadata = None`. You must:

1. Add the import near the top of env.py (after the existing imports):
```python
from trumptrade.core.models import Base  # noqa: F401 — registers all 8 models into Base.metadata
target_metadata = Base.metadata
```

2. Remove or replace the line `target_metadata = None` with `target_metadata = Base.metadata`.

The rest of env.py (the `do_run_migrations`, `run_migrations_online`, `run_async_migrations` functions) comes from the async template and must NOT be changed. Do not hand-roll the async bridge — use the template's implementation verbatim.

Verify the env.py edit is correct:
```bash
python -c "from alembic.config import Config; c = Config('alembic.ini'); print('alembic.ini OK')"
```

**Step 1 — Autogenerate the migration** (run from project root):
```bash
alembic revision --autogenerate -m "initial_schema"
```

This creates a file like `alembic/versions/XXXXXXXXXXXX_initial_schema.py`. The filename prefix is auto-generated (a hex revision ID).

Verify the generated file is NOT empty: open it and confirm `upgrade()` contains `op.create_table(...)` calls for all 8 tables. If `upgrade()` contains only `pass`, the models were not imported — go back to T1 and confirm the `from trumptrade.core.models import Base` import is in env.py.

**Step 2 — Add the app_settings seed to the migration file**: Open the generated migration file. Inside the `upgrade()` function, AFTER all the `op.create_table(...)` calls, add:

```python
    # Seed app_settings defaults (D-10)
    op.execute(
        """
        INSERT INTO app_settings (key, value, updated_at)
        VALUES
            ('position_size_pct',  '2.0',   CURRENT_TIMESTAMP),
            ('stop_loss_pct',      '5.0',   CURRENT_TIMESTAMP),
            ('max_daily_loss_pct', '10.0',  CURRENT_TIMESTAMP),
            ('confidence_threshold', '0.7', CURRENT_TIMESTAMP),
            ('trading_mode',       'paper', CURRENT_TIMESTAMP),
            ('bot_enabled',        'false', CURRENT_TIMESTAMP)
        """
    )
```

Exact key names (must match exactly — downstream phases read these keys by string):
- `position_size_pct` → `2.0`
- `stop_loss_pct` → `5.0`
- `max_daily_loss_pct` → `10.0`
- `confidence_threshold` → `0.7`
- `trading_mode` → `paper`
- `bot_enabled` → `false`

**Step 3 — Run the migration**:
```bash
alembic upgrade head
```

This creates `trumptrade.db` in the project root (or current directory) and applies the migration.

**Step 4 — Verify the database**:
```bash
python -c "
import sqlite3
conn = sqlite3.connect('trumptrade.db')
cursor = conn.cursor()
tables = [r[0] for r in cursor.execute(\"SELECT name FROM sqlite_master WHERE type='table' ORDER BY name\").fetchall()]
print('Tables:', tables)
settings = cursor.execute('SELECT key, value FROM app_settings ORDER BY key').fetchall()
print('Settings:', settings)
conn.close()
"
```

Expected output:
- Tables: `['alembic_version', 'app_settings', 'fills', 'keyword_rules', 'orders', 'posts', 'shadow_portfolio_snapshots', 'signals', 'watchlist']`
- Settings: 6 rows with the seeded keys and values

**If the verification fails:** Delete `trumptrade.db`, fix the issue, then re-run `alembic upgrade head`.
