- Sidebar width: 240px, fixed, never collapses (single-user desktop tool)

- Overall page: `<html class="dark">` — always dark mode, no toggle
- Router: React Router v6 — four routes: `/` (Feed), `/trades`, `/portfolio`, `/settings`

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Tailwind class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, badge inner padding, chip gaps |
| sm | 8px | `p-2` / `gap-2` | Compact row spacing, inline element gaps |
| md | 16px | `p-4` / `gap-4` | Default element spacing, card padding |
| lg | 24px | `p-6` / `gap-6` | Section padding, sidebar nav item padding |
| xl | 32px | `p-8` / `gap-8` | Layout gaps between major sections |
| 2xl | 48px | `p-12` | Major section breaks |
| 3xl | 64px | `p-16` | Page-level outer padding (not used in dense views) |

Exceptions:
- Sidebar nav items: `py-2 px-3` (10px/12px — intentional dense sidebar fit)
- Kill switch button: full sidebar width minus `mx-3`, height `h-9` (36px)
- Table rows: `py-3` (12px) vertical — dense data table standard
- Watchlist ticker chips: `px-2 py-1` (8px/4px) — compact chip convention

## Color

All colors reference the dark-mode CSS custom properties from `index.css .dark {}`.

| Role | CSS Token | OKLCH Approx | Usage |
|------|-----------|-------------|-------|
| Dominant (60%) | `--background` | oklch(0.145 0 0) ≈ #1a1a1a | Page background, main content area |
| Secondary (30%) | `--card` / `--sidebar` | oklch(0.205 0 0) ≈ #222222 | Cards, sidebar, table rows (alternate), expanded rows |
| Accent (10%) | `--primary` | oklch(0.922 0 0) ≈ #e8e8e8 | Kill switch active state ring, focused input border, active nav item indicator |
| Destructive | `--destructive` | oklch(0.704 0.191 22.216) ≈ #e84040 | "Stop Bot" kill switch button background, remove ticker chip X button |

Accent (`--primary`) reserved for:
1. Active sidebar nav item left-border indicator (2px solid)
2. Focused input/select ring on Settings form fields
3. Kill switch button ring when bot is running (pulsing ring animation)

Do NOT use `--primary` for general hover states, headings, or body text emphasis.

## View Contracts

### 1. Sidebar (always rendered)

**Kill Switch Button** (top of sidebar, full-width minus 12px margin each side):
- Bot running: `variant="destructive"` red button, label "Stop Bot", icon `Square`
- Bot stopped: `variant="default"` with green background override (`bg-green-600 hover:bg-green-700 text-white`), label "Start Bot", icon `Play`
- On click: `POST /trading/kill-switch` with `{ "enabled": false }` or `{ "enabled": true }`. Optimistic update — toggle immediately, revert on error.
- Loading state: button `disabled` with spinner icon replacing the action icon

**Nav Items** (Feed, Trades, Portfolio, Settings):
- Layout: icon (16px) + label text, `py-2 px-3`
- Active state: `bg-sidebar-accent text-sidebar-accent-foreground` + 2px `border-l-2 border-primary` left indicator
- Hover state: `bg-sidebar-accent/50`
- Route: React Router `<NavLink>` — active class applied automatically

**Alert Count Badge**:
- Rendered on sidebar below nav, only when `alertCount > 0`
- Red badge: `bg-destructive text-destructive-foreground text-xs px-1.5 py-0.5 rounded-full`
- Label: "N active alert(s)"

**Alert Panel** (below nav, above mode badge):
- Visible only when alerts exist
- Each alert: `<Alert>` component with icon (`AlertTriangle` for warning, `XCircle` for error), short description, timestamp
- Alerts do not auto-dismiss — persist until backend clears them (resolved state from `GET /alerts` polling or WebSocket)
- Panel max-height: `240px` with `ScrollArea` overflow

**PAPER / LIVE Badge** (bottom of sidebar):
- PAPER mode: `bg-yellow-500/10 text-yellow-400 border border-yellow-500/20` rounded badge
- LIVE mode: `bg-red-500/10 text-red-400 border border-red-500/20` rounded badge + pulsing dot animation
- Source: reads `TRADING_MODE` from backend (new `GET /status` endpoint or embedded in existing health check)

Source: CONTEXT.md D-02, D-09, D-10

### 2. Feed Page (`/`)

**Layout**: ScrollArea, full height minus header. New posts prepend at top via WebSocket.

**Post Card**:
```
┌─────────────────────────────────────────────────────────┐
│ [Platform icon 16px]  [relative timestamp — muted]      │
│                                                         │
│ Full post text (body 14px, no truncation)               │
│                                                         │
│ [BULLISH badge]  [0.87]  AAPL  NVDA  TSLA              │
└─────────────────────────────────────────────────────────┘
```

- Card: `bg-card border border-border rounded-lg p-4 mb-2`
- Platform icon: Truth Social → `Globe` (16px), X/Twitter → `Twitter` (16px), both in `text-muted-foreground`
- Timestamp: relative (e.g., "3 min ago") using `date-fns` or browser `Intl.RelativeTimeFormat`, `text-xs text-muted-foreground`
- Post text: `text-sm leading-relaxed text-foreground`
- Sentiment badge: `text-xs font-medium px-2 py-0.5 rounded-full`
  - BULLISH: `bg-green-500/10 text-green-400`
  - BEARISH: `bg-red-500/10 text-red-400`
  - NEUTRAL: `bg-muted text-muted-foreground`
- Confidence: displayed as percentage after badge, `text-xs text-muted-foreground` — e.g., "87%"
- Affected tickers: small `Badge variant="outline"` per ticker, `text-xs`

**Filtered post card**:
- Entire card: `opacity-40`
- No sentiment badge — replace with gray "Filtered" label
- Filter reason shown: `text-xs text-muted-foreground italic` — e.g., "Filtered: too short"
- Source: CONTEXT.md D-05

**WebSocket behavior**:
- Connect on mount to `ws://localhost:8000/ws/feed`
- Reconnect: exponential backoff, 1s → 2s → 4s → 8s → cap at 30s
- On message: prepend new post card to top of list (unshift into React state)
- On disconnect: show inline status chip in feed header: "Reconnecting…" in amber
- Source: CONTEXT.md D-06

**Empty state**:
- Heading: "No posts yet"
- Body: "Waiting for Trump to post. The feed will update automatically."
- Icon: `Zap` at 48px in `text-muted-foreground`

**Loading skeleton**: 4 card skeletons, each 100px tall with shimmer animation via `tw-animate-css`.

### 3. Trades Page (`/trades`)

**Layout**: Full-width table with `ScrollArea` wrapping it.

**Table columns**: Symbol | Side | Qty | Status | Submitted

**Row**:
- Symbol: `text-sm font-medium text-foreground`
- Side: BUY → `text-green-400`, SELL → `text-red-400`, `text-sm`
- Qty: `text-sm tabular-nums text-foreground`
- Status: badge — `submitted` (muted), `filled` (green/10%), `cancelled` (muted), `error` (red/10%)
- Submitted: relative timestamp, `text-xs text-muted-foreground`
- Expand control: `ChevronDown` icon at row right — click entire row to expand

**Expanded row** (Collapsible):
- Background: `bg-secondary/50 border-t border-border`
- Sections rendered vertically with `Separator` between:
  1. **Post**: full post content + platform + original timestamp
  2. **Signal**: sentiment badge, confidence %, affected tickers, final action, reason code, keyword matches (JSON array rendered as tags)
  3. **Fill**: fill price (formatted as `$XX.XX`), filled_at timestamp
  4. **LLM Audit** (Collapsible within expanded row):
     - Trigger: "Show raw LLM prompt / response" link text
     - Content: two `<pre class="text-xs font-mono bg-muted p-3 rounded overflow-auto max-h-48">` blocks, labeled "Prompt" and "Response"
- Source: CONTEXT.md D-07, D-08

**Empty state**:
- Heading: "No trades yet"
- Body: "Trades will appear here once the bot places its first order."
- Icon: `BarChart2` at 48px in `text-muted-foreground`

**Loading skeleton**: 6 row skeletons.

### 4. Portfolio Page (`/portfolio`)

**Summary Cards Row** (3 cards, equal width, `grid grid-cols-3 gap-4`):

| Card | Label | Value format |
|------|-------|-------------|
| Total Equity | "Total Equity" | `$123,456.78` (display size 28px bold) |
| P&L Today | "P&L Today" | `+$234.56` green / `-$234.56` red (display size 28px bold) |
| Buying Power | "Buying Power" | `$12,345.00` (display size 28px bold) |

- Card: `bg-card border border-border rounded-lg p-6`
- Label: `text-xs font-medium text-muted-foreground uppercase tracking-wide`
- Value: `text-3xl font-bold text-foreground mt-2`
- P&L color: positive → `text-green-400`, negative → `text-red-400`, zero → `text-foreground`
- Source: CONTEXT.md D-03

**Positions Table** (below cards, full width):

Columns: Symbol | Qty | Market Value | Avg Entry | Unrealized P&L | % Change

- Market Value, Avg Entry: `tabular-nums`
- Unrealized P&L: colored same as P&L card (green/red)
- % Change: `(value)%` format with same coloring
- Source: CONTEXT.md D-03 (reads live from Alpaca API via `GET /portfolio`)

**Error state** (Alpaca API failure):
- Replace table with: `<Alert variant="destructive">` containing heading "Portfolio unavailable" + body "Unable to reach Alpaca API. Check your credentials and try again."
- Source: CONTEXT.md Claude's Discretion — error boundary handling

**Empty state** (no positions):
- Heading: "No open positions"
- Body: "The bot has no active positions right now."

**Loading skeleton**: 3 card skeletons + 4 row skeletons.

### 5. Settings Page (`/settings`)

**Layout**: Two sections stacked vertically, `gap-8` between them. Each section in a `Card`.

#### Section 1 — Watchlist

- Section heading: "Watchlist" `text-xl font-semibold`
- Description: `text-sm text-muted-foreground` "Add or remove tickers. The bot only trades symbols on this list."
- Chips: `flex flex-wrap gap-2` row of `Badge variant="outline"` components
  - Each chip: ticker symbol + `X` icon (`lucide-react` `X`, size 12px) — clicking X calls `DELETE /watchlist/{symbol}`
  - X button: `text-muted-foreground hover:text-destructive` transition
- Add input row: `Input` (placeholder "Add ticker, e.g. AAPL") + `Button` labeled "Add" with `Plus` icon
  - On submit (Enter key or button click): `POST /watchlist` with `{ "symbol": "AAPL" }`, uppercase the value before send
  - Validation: reject empty input, reject non-alphabetic characters, max 5 chars
  - On error (duplicate, invalid): show `text-destructive text-xs` message below input
- Source: CONTEXT.md D-11

#### Section 2 — Risk Controls

- Section heading: "Risk Controls" `text-xl font-semibold`
- Description: `text-sm text-muted-foreground` "Changes take effect on the next signal."
- Form: 4 labeled `Input` fields in a `grid grid-cols-2 gap-4`

| Field label | Input type | Placeholder | Unit hint |
|-------------|-----------|-------------|-----------|
| Max Position Size | `number` | "2.0" | "% of portfolio" |
| Stop-Loss | `number` | "5.0" | "% from fill price" |
| Max Daily Loss | `number` | "500" | "dollars" |
| Signal Staleness | `number` | "5" | "minutes" |

- Each field: `<label>` above `<Input>` with unit hint in `text-xs text-muted-foreground` below input
- "Save Changes" button: `variant="default"` full-width at bottom, calls `PATCH /settings/risk`
- On save success: show `text-green-400 text-sm` inline "Settings saved." for 3 seconds then hide
- On save error: show `text-destructive text-sm` inline "Failed to save. Try again."
- Source: CONTEXT.md D-11

## Interaction States

| State | Treatment |
|-------|-----------|
| Loading (data fetch) | Skeleton shimmer — never show empty state during initial load |
| Error (fetch failed) | `<Alert variant="destructive">` with specific error copy — never blank screen |
| Empty | Centered icon + heading + body copy — see per-view contract above |
| Optimistic update | Apply immediately, revert on error with `text-destructive` inline message |
| WebSocket reconnecting | Amber status chip in feed header; does not break rest of UI |
| Kill switch loading | Button disabled + spinner; no double-submit |
| Settings save loading | "Save Changes" button disabled + "Saving…" label; inputs remain editable |

## Data Fetching Contract

| Endpoint | Method | TanStack Query key | Stale time | Refetch |
|----------|--------|--------------------|------------|---------|
| `/posts` | GET | `["posts"]` | 30s | On focus |
| `/signals` | GET | `["signals"]` | 30s | On focus |
| `/trades` | GET | `["trades"]` | 30s | On focus |
| `/portfolio` | GET | `["portfolio"]` | 10s | Every 15s (live data) |
| `/watchlist` | GET | `["watchlist"]` | 60s | On focus |
| `/settings/risk` | GET | `["settings", "risk"]` | 60s | On focus |
| `/alerts` | GET | `["alerts"]` | 10s | Every 10s |
| `/trading/kill-switch` | POST | mutation | — | Invalidates nothing |
| `/watchlist` | POST/DELETE | mutation | — | Invalidates `["watchlist"]` |
| `/settings/risk` | PATCH | mutation | — | Invalidates `["settings", "risk"]` |

WebSocket `/ws/feed`: browser native `WebSocket`. Not a TanStack Query concern — managed in a custom React hook `usePostFeed()` that returns `posts: Post[]` and `connectionStatus: "connected" | "reconnecting" | "disconnected"`.
