## Typography

All sizes are Geist Variable. The existing codebase uses `text-xl font-semibold` for page headings and `text-sm` for body/labels — match exactly.

| Role | Size | Tailwind | Weight | Tailwind | Line Height |
|------|------|----------|--------|----------|-------------|
| Display (page heading) | 20px | text-xl | 600 | font-semibold | 1.2 (leading-tight) |

**Code/monospace spans (confirmation phrase):** `font-mono font-semibold text-sm` — used inside modal instruction paragraph to highlight the exact phrase.

### 2. BenchmarksPage

**Route:** `/benchmarks`
**File:** `frontend/src/pages/BenchmarksPage.tsx`

**Data fetching:**

```tsx
const { data, isPending, isError } = useQuery({
  queryKey: ["benchmarks"],
  queryFn: () => api.benchmarks(),
  staleTime: 300_000,      // 5 minutes — updates only at market close
  refetchInterval: 300_000,
})
```

#### 2a. Loading State (skeleton)

While `isPending === true`, render a skeleton placeholder that communicates a chart is loading.

**Exact markup:**

```tsx
<div className="p-6 space-y-4">

      <Skeleton key={s} className="h-4 w-16" />   {/* Legend items */}
    ))}
  </div>
</div>
```

#### 2d. Chart (data loaded, non-empty)

**Component:** `<BenchmarkChart data={data} />` — inline sub-component in BenchmarksPage.

**Full Recharts specification:**

```tsx
<ResponsiveContainer width="100%" height={400}>
  <LineChart data={data} margin={{ top: 8, right: 24, left: 0, bottom: 8 }}>
    <CartesianGrid strokeDasharray="3 3" stroke="hsl(var(--border))" />
    <XAxis
      dataKey="date"
      tick={{ fontSize: 12, fill: "hsl(var(--muted-foreground))" }}
      tickFormatter={(v: string) => v.slice(5)}   // "MM-DD" from "YYYY-MM-DD"
      minTickGap={40}
    />
    <YAxis
      tickFormatter={(v: number) => `${v.toFixed(1)}%`}
      tick={{ fontSize: 12, fill: "hsl(var(--muted-foreground))" }}
      label={{
        value: "Return since start (%)",
        angle: -90,
        position: "insideLeft",
        offset: 12,
        style: { fontSize: 12, fill: "hsl(var(--muted-foreground))" },
      }}
    />
    <ReferenceLine y={0} stroke="hsl(var(--muted-foreground))" strokeDasharray="4 4" />
    <Tooltip
      formatter={(value: number, name: string) => [`${value.toFixed(2)}%`, name.toUpperCase()]}
      labelFormatter={(label: string) => `Date: ${label}`}
      contentStyle={{
        background: "hsl(var(--card))",
        border: "1px solid hsl(var(--border))",
        borderRadius: "0.5rem",
        fontSize: 12,
      }}
    />
    <Legend
      verticalAlign="bottom"
      wrapperStyle={{ paddingTop: 16, fontSize: 12 }}
    />
    <Line type="monotone" dataKey="bot"    stroke="#3b82f6" dot={false} strokeWidth={2} name="Bot" />
    <Line type="monotone" dataKey="spy"    stroke="#22c55e" dot={false} strokeWidth={2} name="SPY" />
    <Line type="monotone" dataKey="qqq"    stroke="#a855f7" dot={false} strokeWidth={2} name="QQQ" />
    <Line type="monotone" dataKey="random" stroke="#f59e0b" dot={false} strokeWidth={2} name="Random" />
  </LineChart>
</ResponsiveContainer>
```

**Chart design decisions:**
- `dot={false}` — no per-point dots; cleaner for daily data over weeks/months
- `type="monotone"` — smooth curve interpolation; standard for financial time-series
- `strokeWidth={2}` — all lines equal weight; Bot is distinguished by color (blue), not thickness
- `minTickGap={40}` — prevents X-axis label collision with many data points
- `ReferenceLine y={0}` — dashed gray 0% baseline; not in legend
- Legend below chart (`verticalAlign="bottom"`) — per D-03
- Y-axis label rotated -90° — standard for percentage-return charts

**Tooltip format:**
- Label row: `Date: 2026-04-23`
- Value rows: `BOT: +1.23%`, `SPY: +0.87%`, `QQQ: +1.10%`, `RANDOM: -0.45%`
- Values formatted to 2 decimal places with `%` suffix
- Series names uppercased in tooltip

**Data type consumed from API:**

```ts
interface BenchmarkPoint {
  date: string    // "YYYY-MM-DD"
  bot: number     // % return, e.g. 1.23 (not 0.0123)
  spy: number
  qqq: number
  random: number
}
```

### 3. LiveModeModal

**File:** `frontend/src/components/LiveModeModal.tsx`
**Trigger:** "Switch to LIVE" / "Switch to PAPER" button in SettingsPage TradingModeSection.

#### 3a. Modal — Enable Live Trading direction

Displayed when current mode is `"paper"` and user clicks "Switch to LIVE Trading".

| Element | Specification |
|---------|---------------|
| Dialog title | "Switch to LIVE Trading" |
| Instruction text | `Type ENABLE LIVE TRADING to confirm.` — phrase displayed in `<code className="font-mono font-semibold">` |
| Input placeholder | `ENABLE LIVE TRADING` |
| Dismiss button | `variant="outline"`, label "Keep PAPER mode" |
| Confirm button | `variant="destructive"`, label "Confirm", `disabled` until `input === "ENABLE LIVE TRADING"` (exact case-sensitive match) |
| Warning text | Below instruction, above input: `"This will use real money. All trades will be placed on your live Alpaca account."` — `text-sm text-muted-foreground` |

#### 3b. Modal — Enable Paper Trading direction

Displayed when current mode is `"live"` and user clicks "Switch to PAPER Trading".

| Element | Specification |
|---------|---------------|
| Dialog title | "Switch to PAPER Trading" |
| Instruction text | `Type ENABLE PAPER TRADING to confirm.` |
| Input placeholder | `ENABLE PAPER TRADING` |
| Dismiss button | `variant="outline"`, label "Keep LIVE mode" |
| Confirm button | `variant="default"`, label "Confirm", `disabled` until `input === "ENABLE PAPER TRADING"` |
| Warning text | None (reverting to paper is safe) |

#### 3c. Interaction flow

```
User clicks switch button
  → modal opens (Dialog open=true)
  → input is empty, Confirm is disabled
  → user types phrase character by character
  → on exact match: Confirm enables
  → user clicks Confirm
    → POST /trading/set-mode { mode, confirmed: true }
    → success: queryClient.invalidateQueries(["portfolio-mode"])
    → modal closes
    → LIVE banner appears / disappears immediately (query refetch)
  → user clicks "Keep PAPER mode" / "Keep LIVE mode" or presses Escape
    → modal closes, no action taken
    → input is reset to "" on next open
```

**Error handling during confirm:** If the API call fails, display an inline error below the input: `"Failed to switch mode. Try again."` using `text-sm text-destructive`. Modal stays open.

**Input state reset:** `useState("")` resets when modal re-opens — clear input in `onOpenChange` handler when transitioning from closed to open (or use `key` prop to unmount/remount the dialog content).

### 4. SettingsPage — Trading Mode Section

**Location:** Added as a third `<Card>` in SettingsPage, below `<RiskControlsSection />`. Separated by `space-y-8` (matches existing section spacing).

**Section structure (matches existing card pattern):**

```tsx
<Card>
  <CardHeader>
    <CardTitle className="text-xl font-semibold">Trading Mode</CardTitle>
    <p className="text-sm text-muted-foreground">
      Switch between paper (simulated) and live (real money) trading.
    </p>
  </CardHeader>
  <CardContent className="space-y-4">
    {/* Current mode display */}
    <div className="flex items-center gap-3">
      <span className="text-sm font-semibold text-foreground">Current mode:</span>
      {mode === "live"
        ? <Badge className="bg-red-500/10 text-red-400 border border-red-500/20 font-semibold text-xs">LIVE</Badge>
        : <Badge className="bg-yellow-500/10 text-yellow-400 border border-yellow-500/20 font-semibold text-xs">PAPER</Badge>
      }
    </div>
    {/* Switch button */}
    <Button
      variant={mode === "live" ? "outline" : "destructive"}
      onClick={() => setModalOpen(true)}
    >
      {mode === "live" ? "Switch to PAPER Trading" : "Switch to LIVE Trading"}
    </Button>
    {/* LiveModeModal */}
    <LiveModeModal
      isLive={mode === "live"}
      open={modalOpen}
      onClose={() => setModalOpen(false)}
    />
  </CardContent>
</Card>
```

**Button variant logic:**
- Paper → Live: `variant="destructive"` (real money at risk — deliberately alarming)
- Live → Paper: `variant="outline"` (safe direction — standard secondary styling)

**Mode source in SettingsPage:** Read from the existing `["portfolio-mode"]` query via `api.portfolio()`. The SettingsPage does not make an additional API call — it shares the cached query result.
